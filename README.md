# Дашборд системных корректировок УУ

Публичный HTML-дашборд для регулярной встречи команды внедрения системных корректировок в направлении управленческого учета.

Рабочая ссылка для просмотра как страницы:

https://htmlpreview.github.io/?https://github.com/akgcapital/htmlki/blob/main/management-accounting-corrections-dashboard.html

> Важно: ссылка GitHub вида `github.com/.../blob/...` показывает HTML как код. Для просмотра дашборда как страницы используйте ссылку через HTMLPreview выше.

## Что внутри

Дашборд состоит из двух вкладок:

1. **Анализ проблематик**
   - группировка выявленных проблематик из обратной связи клиентов;
   - управленческий смысл каждой группы;
   - возможные системные изменения;
   - отметка тем, которые не требуют тяжелого проекта.

2. **Дорожные карты**
   - проекты внедрения системных корректировок по регламенту;
   - общая информация;
   - рабочая группа;
   - этапы и задачи;
   - критерии эффективности;
   - фиксация завершения;
   - план оценки обратной связи через 3-6-9-12 месяцев.

## Файлы

- `management-accounting-corrections-dashboard.html` — основная полная версия дашборда.
- `index.html` — упрощенная публичная версия для быстрого просмотра.

## Как обновить дашборд

1. Откройте файл `management-accounting-corrections-dashboard.html` в GitHub.
2. Нажмите кнопку редактирования файла.
3. Внесите изменения в HTML.
4. Нажмите `Commit changes`.
5. Откройте рабочую ссылку через HTMLPreview.

Если страница показывает старую версию, обновите страницу через `Ctrl + F5` или подождите несколько минут.

## Правила безопасности

- Не загружать в репозиторий исходные Excel-файлы с клиентскими данными.
- Не добавлять названия клиентов, холдингов, ФИО клиентов, договоры, суммы, токены, `.env` и другие приватные данные.
- Для публичной версии использовать только обезличенные формулировки и агрегированные выводы.

## Назначение

Дашборд нужен не для работы с конкретным клиентом, а для управления системными изменениями внутри компании: методология, процессы, качество услуги, клиентская коммуникация и контроль эффективности внедренных корректировок.
